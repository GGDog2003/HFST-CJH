# HH 图像低 PSNR 分析与改进建议

## 1. 先说结论

从 [logs/testlog_14000epoch](D:/python/HFST/HFST-CJH/logs/testlog_14000epoch) 看，`HH` 并不是“整体都差”，而是 **一批早期编号的 HH 病例明显掉点**，把整体 `HH` 平均值拉低了。

- `Guys` 共 119 例，平均 `32.4103 dB / 0.9485`
- `HH` 共 53 例，平均 `28.5017 dB / 0.9184`
- 但 `HH` 内部明显分成两群：
- `HH 1211-1359` 共 19 例，平均 `19.1126 dB / 0.8522`
- `HH 1360-1399` 共 4 例，平均 `35.6750 dB / 0.9640`
- `HH 1400+` 共 30 例，平均 `33.4917 dB / 0.9542`

所以更准确的说法不是“HH 大部分都只有十几 dB”，而是：

**HH 里一段特定子域（主要是 1211-1359 这批）和模型/训练分布严重不匹配，导致 PSNR 集体崩掉。**

## 2. 日志里的直接证据

低分 `HH` 样本高度集中，典型如下：

- `IXI093-HH-1359-T2.nii.gz | 15.47dB`
- `IXI048-HH-1326-T2.nii.gz | 16.35dB`
- `IXI094-HH-1355-T2.nii.gz | 16.63dB`
- `IXI083-HH-1357-T2.nii.gz | 17.23dB`
- `IXI033-HH-1259-T2.nii.gz | 17.57dB`
- `IXI039-HH-1261-T2.nii.gz | 17.91dB`

但后面的不少 `HH` 又恢复正常甚至很好：

- `IXI126-HH-1437-T2.nii.gz | 36.69dB`
- `IXI146-HH-1389-T2.nii.gz | 37.35dB`
- `IXI150-HH-1550-T2.nii.gz | 35.54dB`
- `IXI159-HH-1549-T2.nii.gz | 34.54dB`
- `IXI195-HH-1620-T2.nii.gz | 34.07dB`

这说明问题不是“模型完全不会做 HH”，而是 **模型不会处理 HH 里的某一类采集风格/强度风格**。

## 3. 为什么会这样

### 3.1 最可能的主因：HH 内部存在子域偏移

从日志编号看，低分样本几乎都集中在 `HH-1211` 到 `HH-1359`。而 `HH-1388` 以后大量样本恢复到 `31-37 dB`。

这非常像下面几类问题之一：

- 同一个医院 `HH` 内部实际用了两套采集协议
- 早期病例和后期病例的强度分布、对比度、噪声水平不同
- T1/T2 的跨模态对应关系在早期那批样本上更弱
- 训练集中这类子域样本占比太少，模型学到的是主流域

也就是说，`HH` 不是一个单一分布，而是混了至少两个分布。模型在主流分布上表现正常，在早期 `HH` 子域上明显失配。

### 3.2 数据强度分布确实有异常迹象

我抽查了几例低分和高分 `HH` 的原始体数据统计，发现低分早期 `HH` 的原始强度量级明显偏高。

示例：

- `IXI033-HH-1259`
- `T1 mean/std/p99 = 309.338 / 480.634 / 1694.049`
- `T2 mean/std/p99 = 119.631 / 196.275 / 964.035`
- `IXI093-HH-1359`
- `T1 mean/std/p99 = 351.135 / 530.383 / 1834.635`
- `T2 mean/std/p99 = 120.946 / 212.890 / 1156.940`

对比高分样本：

- `IXI095-HH-1390`
- `T1 mean/std/p99 = 110.700 / 182.340 / 640.000`
- `T2 mean/std/p99 = 61.874 / 112.306 / 546.000`
- `IXI126-HH-1437`
- `T1 mean/std/p99 = 122.109 / 174.367 / 594.000`
- `T2 mean/std/p99 = 104.193 / 164.367 / 846.000`

这说明早期低分 `HH` 很可能不是单纯“难一点”，而是 **原始强度分布和后期/主流样本明显不同**。

### 3.3 当前归一化方式可能放大了这种域偏移

在 [data/dataset_sr.py](D:/python/HFST/HFST-CJH/data/dataset_sr.py) 的 `DatasetIXIT` 中，T1 与 T2 是这样处理的：

- 先读取整卷 `T1` 和 `T2`
- 再用 **两者联合的全局最大最小值** 做归一化
- 然后对退化后的 LR 结果再单独做一次归一化

这会带来两个风险：

- 不同医院/协议下，只要某一模态的极值特别大，就会一起影响另一模态的缩放比例
- 低分子域里如果强度离群更明显，T1 参考模态和 T2 目标模态的相对对比度关系会被扭曲

对于跨模态超分，这种问题比单模态更敏感，因为模型同时依赖：

- `L[0]`：退化后的低分辨率 T2
- `L[1]`：高分辨率 T1 参考

一旦某个子域里 **T1/T2 的缩放关系变形**，参考引导就会变得不稳定，PSNR 很容易整体掉下去。

### 3.4 跨模态参考本身对域偏移非常敏感

当前任务不是普通 SISR，而是“低分辨率 T2 + 高分辨率 T1 参考 -> 重建高分辨率 T2”。

这类方法默认一个前提：

- T1 和 T2 在结构上高度对应
- 不同病例之间这种对应关系比较稳定

但早期低分 `HH` 子域如果存在：

- 对比反转更强
- 脑组织边界强度关系不同
- 噪声/平滑风格不同

那么参考模态带来的就不一定是帮助，可能反而是错误引导。这样模型就会在整卷切片上持续犯错，所以出现成批 `15-22 dB` 的结果。

### 3.5 评估方式可能会放大问题，但不是根因

在 [main_test_spsr.py](D:/python/HFST/HFST-CJH/main_test_spsr.py) 中，PSNR 是对整卷 `minn:maxn` 全部切片直接求的，`minn=0, maxn=156`。

这带来两个次要风险：

- 如果某些卷前后景很少、背景很多，整卷平均会受影响
- 如果某些子域在边缘切片上的配准或强度更不稳定，整体分数会被进一步拉低

但因为问题高度集中在 `HH-1211~1359`，而不是所有样本都这样，所以这更像是“放大器”，不是核心原因。

## 4. 我认为最可能的原因排序

### 第一优先级

**HH 早期病例（1211-1359）与训练主分布存在明显域偏移。**

这是最强解释，因为它同时符合：

- 低分集中在一个编号区间
- 同医院后期病例又恢复正常
- 原始强度统计存在明显差异

### 第二优先级

**联合归一化 + LR 单独再归一化，使跨模态强度关系在异常子域里进一步失真。**

这会直接伤害 T1 参考对 T2 重建的帮助作用。

### 第三优先级

**跨模态参考模型对异常协议的泛化不足。**

也就是模型学会了“常规 T1/T2 对应关系”，但没有学会“协议变化后仍然稳定对齐”。

### 第四优先级

**整卷评估和切片范围设置会放大问题。**

它解释不了“为什么只在一批 HH 上崩”，但会让崩得更明显。

## 5. 改进建议

下面按投入产出比排序。

### 建议 1：先按子域拆分统计，不要只看 HH 总均值

先把测试结果至少拆成：

- `Guys`
- `HH-1211~1359`
- `HH-1360+`

如果拆开后只有早期 `HH` 明显掉点，那后续训练和评估都应该把它当成独立子域，而不是继续把所有 `HH` 混在一起。

这是最先该做的，因为它几乎不需要改模型，就能确认问题边界。

### 建议 2：检查训练集里早期 HH 子域的占比

重点看：

- `HH-1211~1359` 这类样本在训练集里有没有
- 数量是不是明显少于 `Guys`
- 是否只在测试集出现，而训练集几乎没有

如果这类样本训练时很少见，当前结果就非常合理，本质上就是测试时遇到了未覆盖子域。

### 建议 3：把归一化改成更稳的医学影像方案

优先考虑：

- 每个模态各自独立归一化，不要用 T1/T2 联合 min-max
- 用百分位裁剪后再归一化，例如 `0.5%~99.5%`
- 只在前景脑区内统计归一化范围，避免背景和离群值主导缩放
- 保证 HR/LR/Ref 使用一致且可解释的强度标准

这条建议我认为收益会很大，因为当前低分子域最明显的异常就是强度分布不一致。

### 建议 4：为 HH 早期子域做单独的数据增强或重采样

可以考虑：

- 训练时提高早期 `HH` 样本采样权重
- 对训练数据加入更强的强度扰动、gamma、bias field、噪声增强
- 对参考模态和目标模态分别做更丰富的对比度扰动

目的不是“记住那几例”，而是让模型学会对协议变化更鲁棒。

### 建议 5：增加域自适应/域稳健设计

如果后面还想继续做模型层优化，可以考虑：

- 在跨模态融合前增加更强的模态对齐模块
- 引入 instance normalization / adaptive normalization 一类的域稳健手段
- 在特征层做 domain adversarial 或 domain-specific branch
- 先做粗配准/风格标准化，再做跨模态超分

这类改动成本更高，但如果目标是“跨医院稳定泛化”，是值得考虑的。

### 建议 6：把评估拆成前景区和全图两套指标

建议补充：

- 全图 PSNR/SSIM
- 脑区 mask 内 PSNR/SSIM
- 去掉明显空白边缘切片后的 PSNR/SSIM

这样可以判断问题到底主要出在：

- 脑组织重建本身
- 还是背景/边缘/空切片把均值拉低

### 建议 7：单独做“是否使用 T1 参考”的消融

建议把早期低分 `HH` 子域单独做一组对比：

- 只用 LR T2 做 SISR
- LR T2 + T1 参考做当前方法

如果在早期 `HH` 上“加 T1 参考反而更差”，就说明不是超分主干坏了，而是 **参考模态在异常子域中提供了错误引导**。

这个实验对定位问题非常关键。

## 6. 推荐的排查顺序

建议按下面顺序推进，效率最高：

1. 先把测试结果按 `HH-1211~1359 / HH-1360+ / Guys` 分组统计
2. 检查训练集中是否覆盖了 `HH-1211~1359` 这类样本
3. 对这批低分样本画出 T1/T2 直方图、切片可视化、归一化后对比图
4. 做“不使用 T1 参考”的消融
5. 再决定是优先改归一化，还是优先改训练采样与增强

## 7. 最终判断

我目前的判断是：

**HH 图像出现大量十几 dB 的主要原因，不是 HH 全部难，而是 HH 早期一批病例形成了明显的子域偏移；当前数据归一化和跨模态参考机制对这个子域不够稳健，最终导致整批样本 PSNR 集体偏低。**

如果只允许优先做一件事，我最建议先做：

**把 HH 按编号子域拆开统计，并检查训练集覆盖情况。**

因为从现有证据看，这一步最有可能直接解释问题。
